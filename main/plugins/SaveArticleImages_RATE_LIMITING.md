# SaveArticleImages 插件 - API频率限制功能

## 功能概述

SaveArticleImages 插件现在支持API图床调用的频率限制功能。当文章包含大量图片时，系统会自动对API调用进行限速，确保不会超过配置的最大调用频率，避免触发API服务的频率限制错误。

## 核心特性

### 1. 令牌桶算法限流
- 使用标准的令牌桶算法实现平滑的频率限制
- 默认每分钟最多20次API调用（可配置）
- 令牌以均匀速率补充，避免突发流量

### 2. 异步队列处理
- 超出频率限制的上传任务自动进入队列等待
- 使用工作池模式处理队列任务
- 支持配置最大队列长度，防止内存溢出

### 3. 智能上传策略
- 优先尝试直接上传（有可用令牌时）
- 直接上传失败后自动转入队列
- 队列满时返回友好的错误提示

### 4. 实时统计与监控
- 提供队列长度、工作池状态等实时统计
- 详细的日志记录，便于调试和监控
- 支持获取当前频率限制状态

## 配置参数

### 新增配置项

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `api_rate_limit_per_minute` | int | 20 | API每分钟最大调用次数 |
| `api_max_queue_size` | int | 1000 | 上传队列最大长度 |
| `api_queue_timeout` | int | 300 | 队列任务超时时间（秒） |

### 配置界面

在插件配置的"图床API"标签页中，新增了"频率限制设置"区域：

- **每分钟调用限制**：控制API的最大调用频率
- **队列最大长度**：限制等待上传的任务数量
- **队列任务超时**：设置任务在队列中的最长等待时间

## 工作流程

### 正常流程
1. 文章保存时触发图片处理
2. 检测到远程图片需要上传到API图床
3. 检查频率限制器是否有可用令牌
4. 有令牌：直接执行上传
5. 无令牌：将任务加入队列，等待处理

### 队列处理流程
1. 队列处理器每100ms检查一次
2. 有可用令牌时，从队列取出任务
3. 提交到工作池执行上传
4. 上传完成后返回结果
5. 更新文章中的图片URL

### 错误处理
- 队列满时：返回"上传队列已满"错误
- 任务超时：返回"上传任务超时"错误
- 上传失败：记录日志，不影响其他任务

## 性能优化

### 1. 工作池模式
- 默认5个工作协程并行处理
- 非阻塞任务提交
- 支持动态调整工作线程数

### 2. 内存管理
- 限制队列最大长度
- 及时清理完成的任务
- 使用对象池减少GC压力

### 3. 上下文管理
- 支持优雅关闭
- 超时控制
- 资源清理

## 监控与调试

### 日志记录
插件会记录以下关键事件：
- 频率限制器初始化
- 上传队列初始化
- 任务加入队列
- 任务开始处理
- 任务完成（成功/失败）
- 队列处理器状态变化

### 统计信息
可通过 `GetQueueStats()` 方法获取：
- 当前队列长度
- 队列容量
- 频率限制配置
- 工作池状态
- 可用令牌状态

## 注意事项

1. **API配置要求**：使用频率限制功能时，必须正确配置API上传地址
2. **队列长度**：根据服务器内存和预期并发量合理设置队列长度
3. **超时时间**：队列超时时间应大于预期的最大等待时间
4. **监控建议**：建议定期查看队列统计信息，及时调整配置

## 故障排除

### 常见问题

1. **上传队列已满**
   - 增加 `api_max_queue_size`
   - 检查API服务是否正常
   - 降低上传频率或增加工作线程

2. **任务超时**
   - 增加 `api_queue_timeout`
   - 检查网络连接
   - 确认API服务响应时间

3. **频率限制不生效**
   - 确认 `upload_target` 设置为 "api"
   - 检查 `api_rate_limit_per_minute` 是否大于0
   - 查看日志确认频率限制器已初始化

### 调试建议

1. 开启详细日志记录
2. 使用 `GetQueueStats()` 监控队列状态
3. 逐步调整频率限制参数
4. 测试不同并发场景
